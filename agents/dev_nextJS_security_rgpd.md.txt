---

## Next.js Engineering Standards

- Use **App Router exclusively**
- Default to **Server Components**
- Use Client Components only when strictly necessary
- Prefer:
  - Server Actions over API routes (when appropriate)
  - co-location of logic
- Enforce clean structure:
  - `/app` → routing
  - `/features` or `/modules` → business logic
  - `/lib` → shared utilities
- Optimize for:
  - performance (SSR / streaming)
  - minimal JS shipped to client

---

## Security by Design (MANDATORY)

- NEVER trust client input — validate everything server-side
- Use:
  - schema validation (e.g. Zod or equivalent)
  - strict typing everywhere
- Enforce:
  - authentication checks in all sensitive actions
  - authorization (role/ownership checks)
- Prevent:
  - XSS (no unsafe HTML injection)
  - CSRF (use proper patterns with server actions)
  - injection attacks
- Secrets:
  - NEVER exposed to client
  - ONLY via secure env handling
- Logs must NOT expose sensitive data

---

## RGPD / Privacy by Design (MANDATORY)

- Only collect strictly necessary data
- For every stored data:
  - define purpose
  - define retention
- Implement:
  - explicit consent handling when required
  - ability to delete user data
- NEVER log:
  - personal data
  - tokens
  - sensitive payloads
- Ensure:
  - data isolation per user
  - secure API boundaries

---

## Testing & Quality (STRICT)

- **Unit tests are mandatory for every feature**
- Follow TDD when possible:
  1. write failing test
  2. implement
  3. refactor
- Cover:
  - business logic
  - edge cases
- Use:
  - mocks for external dependencies
- Minimum expectations:
  - deterministic tests
  - no flaky tests
- Run full test suite after EACH task
